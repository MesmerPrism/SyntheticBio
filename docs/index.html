What it is for

Built for repeatable demos, test runs, and fixture export.

SyntheticBio is the operator-side companion app. It runs the live named-pipe server, exposes documentation-friendly virtual devices, exports offline fixture sets, and helps launch PolarH10 in synthetic transport mode on the matching pipe.

Research Preview

The downloadable preview is self-signed. The guided setup helper handles the certificate trust step for you, then opens App Installer.

Why use it

Demo the app reliably

Run coherence, HRV, and breathing-dynamics showcase scenarios with the same timings every time instead of depending on a live operator session.

Export fixture bundles

Write deterministic scenario folders with ground truth, analysis traces, RR/HR CSV, ECG CSV, and session metadata for documentation and validation.

Test same-machine integration

Launch or reuse PolarH10 against the synthetic transport so the Windows app exercises its real decode and tracker path while the input side stays controlled.

Typical workflow

01

Start SyntheticBio

Open the operator app, keep the default pipe unless you have a reason to change it, and start the live server or press Establish connection.

02

Open PolarH10 on synthetic transport

Let SyntheticBio launch it, or open PolarH10 yourself with --transport synthetic --synthetic-pipe polarh10-synth.

03

Scan and connect

Use the virtual devices in PolarH10 just like live straps. The scenarios loop, which keeps longer validation and documentation sessions practical.

04

Export when needed

Write deterministic fixtures and benchmark runs from SyntheticBio when you need reusable artifacts instead of a live collection session.
